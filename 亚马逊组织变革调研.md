# 亚马逊组织变革调研

> 整理时间：2026-05-12｜来源：Reuters / Fortune / Seattle Times / CNBC / 新浪财经

---

## 一、组织架构：强制压扁管理层

- **硬指标下达**：2024年9月 Jassy 发全员信，要求各 S-Team 在 2025年Q1 前将 IC/Manager 比例提升 ≥15%。
- **执行方式**：合并团队、Manager 降级转 IC 为主，未大规模裁员即达标。
- **随后清算存量**：2025年10月裁 1.4 万、2026年1月再裁 1.6 万，白领合计裁员约 3 万（占白领总数 ~10%），重点砍 AWS、零售、HR 中层。
- **官方定性**：内部文件明确目标是"减少层级、增加 ownership、去除官僚主义"。

## 二、职级体系：L 制硬墙 + Builder 头衔试验

### L 级结构（未取消）

| 级别 | 对应角色 | 软件工程师 TC 参考 |
|------|---------|-----------------|
| L4 | 初级 / 新人 | $180k–$220k |
| L5 | 中级 SDE | $240k–$320k |
| L6 | 高级 SDE（多数人上限） | $280k–$400k |
| L7 | Staff | $450k+ |
| L8 | Principal | $700k+ |
| L10 | Director | $1M+ |

- **Band 是硬墙**：薪酬上限由级别决定，不是参考线，定级即定天花板。
- **IC 通道打通至 L10**：技术专家不做管理也能走到 Principal，TC 与管理通道持平。
- **2025 年 Band 收紧**：内部薪酬公平审计后，高通胀年份超额授薪被收回，各级上限下调。

### Builder 头衔试验（2026年4月最新）

- **发起人**：Ring + Blink 事业部 CPO Jason Mitura，2026年4月23日媒体报道。
- **具体做法**：取消 PM / SDE / Senior / Lead 等传统职位名，产品相关白领统称 **Builder**，管理者叫 **Builder Lead**。
- **影响范围**：目前仅限 Ring、Blink 数百名白领，从 2026年5月考核季开始落地。
- **薪酬影响**：官方明确表态"薪酬、晋升路径不变"，L 级硬墙和薪酬 Band 照旧，只是对外不显示职位前缀。
- **员工顾虑**：去掉级别头衔后，外部跳槽时资历如何证明？

> **两者关系**：L 级没消失，只是藏起来了；Builder 是"对外和评价语言"的变化，L 级是"薪酬和晋升"的底层逻辑，并行存在。

## 三、激励体系：RSU 主导 + 新增现金转换

- **RSU 前轻后重，强制留人**：归属比例第1年5%、第2年15%、第3–4年每半年20%，前两年离职代价极高。
- **2024 年起 25% RSU 可转现金**：L4–L8 员工可将当期归属 RSU 按固定价格（2025年定价 $217.59/股）换成现金，解决股价集中风险。
- **归属频率从半年改为季度**：5月/8月/11月/2月，流动性提升，降低优秀员工因等待归属而被迫留守的摩擦。
- **底薪上限已提至 $350k**：2022年改，应对高通胀期人才竞争。

## 四、考核体系：三层嵌套，强制分布

### 工具一：Forte（年度评估系统，2025年升级）

- 员工必须提交 **3–5 条具体成就**，不接受泛泛自述。
- 管理者三维打分：**绩效表现 × 发展潜力 × LP践行度**（领导力原则）。
- LP 维度是 2025 年新加入的，历史上首次量化进考核；**顶级"Role Model"评级仅 5% 员工可得**。
- 三维综合为 OV 分（Overall Value），决定涨薪 / 晋升 / PIP。

### 工具二：OLR（组织领导力评估）

- 形式上不叫 stack ranking，实际按人头相对排序。
- Manager 被分配强制分布配额，即使团队整体优秀也须排出底部。

### 工具三：URA（不遗憾流失率）

- 公司每年设定约 **6% 员工离开**的目标（走了也不遗憾）。
- 自然离职达标则不用动手，否则推人进 PIP——将"主动淘汰"合规化、常态化。

### 工具四：PIP（绩效改进计划）

- 第一步叫 **"Focus"**（员工通常不被正式告知，但内部皆知）。
- Focus → PIP → 终止，约 **70% 进入者最终离职或被解雇**。

### 考核逻辑的方向性转变（Builder 试验的信号）

- 旧逻辑：你是什么岗位、什么级别 → 对应什么考核标准。
- 新逻辑：**你为多大规模、多少层级的客户创造了多少价值** → 才是真正的评价起点。

## 五、背后逻辑主线

| 问题 | 亚马逊的解法 |
|------|------------|
| 层级太多、决策慢 | 强制 IC/Manager 比≥15%，Manager 降级/转岗 |
| 职位名阻碍协作和跨界 | Builder 单一职位族，去掉 Senior/Lead 前缀 |
| 优秀 IC 不愿做管理 | IC 通道打通至 L10，TC 不低于管理通道 |
| 大公司留不住好人 | RSU 转现金 + 季度归属，降低离职摩擦 |
| 怎么自然淘汰末位 | URA 目标 → OLR 排名 → PIP，让"走人"成为系统输出 |
| AI 转型期需降本增效 | 收紧 Band + 大规模裁撤中层 + 释放 headcount 给 AI 岗 |
